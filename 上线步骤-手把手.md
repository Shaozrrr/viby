# Viby 上线到 viby.ink：手把手（在哪个网站、怎么填）

按顺序做即可。只有 **「你的网站、你的服务器」** 我无法替你登录，其余字段都按下面抄。

---

## 开始前：确认两件事

1. 域名 **viby.ink** 是在 **腾讯云** 买的，还是在 **阿里云 / Namecheap** 等别处买的？  
   - 只在最后 **解析 DNS** 时进对应网站即可，步骤里两种都会写。
2. 需要一台 **Linux 云服务器**（建议 Ubuntu），用来跑 Node + Nginx。

---

# 第一步：在 GitHub 创建 OAuth 应用（必须先做，后面要填密钥）

## 1.1 打开正确页面

**如果你看到 404：** 多半是地址栏里多复制了符号。  
例如误把中文括号 **`）`** 粘到了链接后面，浏览器会变成 `...developers%EF%BC%89`，这就是 404。  
请只复制下面**代码框里这一行**（不要带任何中文标点、不要带星号、不要带后面的说明文字）：

```
https://github.com/settings/developers
```

**更省事（直接打开「新建 OAuth 应用」表单，同样需要已登录 GitHub）：**

```
https://github.com/settings/applications/new
```

### 方式 A：用链接（推荐）

1. 先打开 **https://github.com** 并 **登录账号**（未登录时设置页可能异常）。
2. 在地址栏 **完整替换** 为上面两个链接之一，回车。

### 方式 B：只靠点击（不记网址）

1. 浏览器打开：**https://github.com/login** 并登录。
2. 右上角 **头像** → **Settings**（设置）。
3. 左侧菜单 **滑到最底部** → **Developer settings**。
4. 点 **OAuth Apps**。
5. 点 **New OAuth App**。

## 1.2 表单怎么填（逐行对照）

| 页面上的英文名称 | 填什么 |
|------------------|--------|
| **Application name** | 随意，例如：`Viby` |
| **Homepage URL** | `https://viby.ink` |
| **Application description** | 可留空 |
| **Authorization callback URL** | **必须完全一致**：`https://viby.ink/api/auth/github/callback` |

> 注意：必须是 **`https`**，末尾**没有多余斜杠**，路径必须是 **`/api/auth/github/callback`**。

6. 点 **Register application**（注册应用）

## 1.3 拿到两个密钥（先复制到备忘录）

注册成功后你会在应用详情页看到：

1. **Client ID**  
   - 一长串字符，旁边一般可以复制。→ **复制保存**，待会写到服务器上的 `.env` 里。

2. **Client secrets**  
   - 点 **Generate a new client secret**（生成新密钥）  
   - 生成后 **立刻复制**（通常只显示一次）→ 保存为 **Client secret**  
   - 待会写到服务器的 `.env` 里的 `GITHUB_CLIENT_SECRET`。

# 第二步：在腾讯云买服务器并拿到公网 IP

## 2.1 打开腾讯云控制台

1. 打开：**https://console.cloud.tencent.com/**
2. 微信 / QQ / 邮箱 登录你的腾讯云账号。

## 2.2 购买轻量应用服务器（新手推荐）

1. 顶部搜索框输入 **轻量应用服务器**，或打开：  
   **https://console.cloud.tencent.com/lighthouse**
2. 点 **新建** / **购买**
3. 选择：
   - **地域**：离你或用户近的（例如广州、上海）
   - **镜像**：**Ubuntu 22.04**（或 20.04）
   - **套餐**：选最低配置即可
4. 设置 **实例密码**（或用 SSH 密钥），记住密码
5. 付款，等待创建完成

## 2.3 记录公网 IP

1. 在轻量控制台 → 点击你的 **实例名称**
2. 在概要里找到 **IP 地址** 里的 **公网 IPv4**，例如 `123.45.67.89`  
   → **复制保存**，下面解析域名要用。

## 2.4 放行 80、443、22 端口（否则网站和 SSH 进不去）

1. 仍在该实例页面，找到 **防火墙**（有的叫 **安全组**，轻量一般在「防火墙」页签）
2. **添加规则**（若 22 已有一条就不用再管）：
   - **TCP 80**，来源 `0.0.0.0/0`
   - **TCP 443**，来源 `0.0.0.0/0`
   - **TCP 22**（若列表没有就加），来源可先 `0.0.0.0/0`（仅学习用；正式可改为你家宽带 IP）
3. 保存

# 第三步：把域名 viby.ink 指到你的服务器

## 情况 A：域名在腾讯云（DNSPod）

1. 打开：**https://console.dnspod.cn/**  
   或在腾讯云控制台搜索 **DNS解析 DNSPod**：**https://console.cloud.tencent.com/cns**
2. 在域名列表里找到 **viby.ink** → 点 **解析**
3. 点 **添加记录**：
   - **主机记录**：填 `@`（表示 `viby.ink` 本身）
   - **记录类型**：**A**
   - **记录值**：填第二步复制的 **公网 IP**
   - **TTL**：默认即可（如 600）
4. 保存

（可选）想让 **www.viby.ink** 也能访问：

- 再 **添加记录**：主机记录填 **`www`**，类型 **A**，记录值 **同一个公网 IP**

> 生效可能要 **几分钟到几小时**。可以用手机流量访问 `https://viby.ink` 试（避免本地 DNS 缓存）。

## 情况 B：域名在别的注册商（阿里云、GoDaddy 等）

1. 登录你 **买 viby.ink 的网站**
2. 找到 **域名管理** → **DNS 解析** / **DNS Management**
3. 添加一条：
   - **类型**：A  
   - **主机名**：`@` 或留空（看服务商界面提示）
   - **值**：你的 **公网 IP**
4. 保存；有 `www` 需求就再加一条主机名 `www` 的 A 记录，指向同一 IP。

# 第四步：用 SSH 登录你的服务器

在你自己的电脑上打开终端：

- **Mac / Linux**：打开「终端」
- **Windows**：打开 **PowerShell**，或安装 **Windows Terminal**

把下面命令里的 `你的公网IP` 换成真实 IP：

```bash
ssh root@你的公网IP
```

若提示用户名不对，腾讯云 Ubuntu 镜像常见用户是 **`ubuntu`**，则：

```bash
ssh ubuntu@你的公网IP
```

第一次会问 `yes/no`，输入 `yes`，再输入你在购买服务器时设的 **密码**。

登录成功后，后面命令都在 **服务器里** 执行。

---

# 第五步：在服务器上安装 Node、Nginx、PM2（复制执行）

在 **SSH 已登录** 的窗口里，一段段复制执行：

```bash
sudo apt update
sudo apt install -y nginx git curl
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
sudo npm install -g pm2
node -v
```

看到 `v20.x` 之类即成功。

---

# 第六步：把网站代码放到服务器

**任选一种你喜欢的方式。**

## 方式 A：你有 Git 仓库（推荐）

把下面地址换成你的仓库：

```bash
sudo mkdir -p /var/www
cd /var/www
sudo git clone https://github.com/你的用户名/你的仓库名.git viby
sudo chown -R $USER:$USER /var/www/viby
cd /var/www/viby
```

## 方式 B：没有 Git，从你自己电脑上传

在你 **自己电脑**（不是服务器）打开终端，进入你放项目的文件夹上层，执行（把 IP 换成公网 IP）：

```bash
scp -r ./empty-window ubuntu@你的公网IP:/home/ubuntu/viby
```

然后在 **服务器** 上：

```bash
sudo mv /home/ubuntu/viby /var/www/viby
cd /var/www/viby
```

（若你是 `root` 用户，路径可能是 `/root/viby`，则 `cd` 到那里即可。）

# 第七步：配置 `.env`（把 GitHub 的密钥写进去）

在服务器上（路径按你实际，下面以 `/var/www/viby` 为例）：

```bash
cd /var/www/viby
cp .env.example .env
nano .env
```

用键盘编辑，保证 **至少有这些行**（把中文换成你第一步复制的真实值）：

```env
PORT=4184
HOST=127.0.0.1
PUBLIC_ORIGIN=https://viby.ink
GITHUB_CLIENT_ID=这里粘贴ClientID
GITHUB_CLIENT_SECRET=这里粘贴ClientSecret
```

保存：**按 `Ctrl+O` 回车**，退出：**按 `Ctrl+X`**。

---

# 第八步：启动网站进程（PM2）

```bash
cd /var/www/viby
bash scripts/pm2-start.sh
```

自检：

```bash
curl -s http://127.0.0.1:4184/api/health
```

应看到 JSON 里有 `"ok":true`，且 `"githubOAuthReady":true`。

---

# 第九步：Nginx 反代 + HTTPS（两种做法选一种）

你需要 **SSL 证书**。新手推荐 **Let’s Encrypt（免费）**。

## 9.1 安装 Certbot（在服务器上）

```bash
sudo apt install -y certbot python3-certbot-nginx
```

## 9.2 先放一个简单 Nginx 站点，只为申请证书

```bash
sudo tee /etc/nginx/sites-available/viby.ink <<'EOF'
server {
    listen 80;
    server_name viby.ink www.viby.ink;
    location / {
        proxy_pass http://127.0.0.1:4184;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
EOF
sudo ln -sf /etc/nginx/sites-available/viby.ink /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t && sudo systemctl reload nginx
```

## 9.3 申请证书（邮箱换成你的真实邮箱）

```bash
sudo certbot --nginx -d viby.ink -d www.viby.ink --email 你的邮箱@example.com --agree-tos --no-eff-email
```

按提示选 **Redirect HTTP to HTTPS**（重定向到 HTTPS）。

Certbot 会自动改 Nginx；请确认配置里 **仍然有**：

```nginx
proxy_set_header X-Forwarded-Proto $scheme;
```

若没有，用 `sudo nano /etc/nginx/sites-available/viby.ink` 在 `location /` 里补上，然后：

```bash
sudo nginx -t && sudo systemctl reload nginx
```

## 9.4 如果你不用 Certbot，而用腾讯云下载的证书

把证书文件放到服务器某个目录后，可参考仓库里的  
`deploy/nginx-viby.ink.example.conf`  
修改 `ssl_certificate` / `ssl_certificate_key` 两行路径，再：

```bash
sudo cp deploy/nginx-viby.ink.example.conf /etc/nginx/sites-available/viby.ink
# 编辑证书路径后再：
sudo ln -sf /etc/nginx/sites-available/viby.ink /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

# 第十步：浏览器验收

1. 打开：**https://viby.ink**
2. 打开：**https://viby.ink/api/health**
3. 点页面 **登录** → **使用 GitHub 登录**，应能授权并回到首页，右上角显示昵称。

若 GitHub 报错 **redirect_uri**，回到 **GitHub → Settings → Developer settings → OAuth Apps → 你的应用**，核对 **Authorization callback URL** 是否 **一字不差**：

`https://viby.ink/api/auth/github/callback`

---

# 常见卡住：对照这一张表

| 现象 | 检查 |
|------|------|
| SSH 连不上 | 防火墙是否放行 **22**；IP 是否对；用户名是 root 还是 ubuntu |
| 浏览器打不开 | 是否 **DNS 已生效**；防火墙 **80/443**；Nginx 是否 `systemctl status nginx` |
| `/api/health` 失败 | 服务器上 `pm2 logs viby`；`curl http://127.0.0.1:4184/api/health` |
| GitHub 登录失败 | 回调 URL；`.env` 里 `PUBLIC_ORIGIN` 是否为 `https://viby.ink` |
